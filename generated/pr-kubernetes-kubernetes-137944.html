Immediate Shutdown for Scheduler Metrics Async Recorder

This PR improves the shutdown behavior of the Kubernetes scheduler's default metrics async recorder. Previously, the recorder's background goroutine could only stop after a sleep interval, causing delays in test execution. The update allows the recorder to stop immediately, significantly reducing test runtimes.

Key Changes

  • Refactored the async recorder's goroutine to check for shutdown signals before sleeping.
  • Recorder now flushes metrics and checks for stop signals using select with time.After.
  • Unit tests added to verify correct buffering, flushing, and immediate shutdown behavior.
  • Test runtime for DynamicResources plugin reduced from ~91s to ~13s.
  • No user-facing changes; internal cleanup and performance improvement.

File Changes

  • pkg/scheduler/metrics/metric_recorder.go — modified
    Refactored the MetricAsyncRecorder's run loop to check for stop signals before sleeping, enabling immediate shutdown.
  • pkg/scheduler/metrics/metric_recorder_test.go — modified
    Added and updated unit tests to validate the new shutdown behavior and ensure correct metric buffering and flushing.

Risks & Impact

  • Potential for missed or duplicated metric flushes if the new select logic is incorrect.
  • Possible unforeseen side effects in production if the shutdown logic interacts with other scheduler components.
  • Unit tests may not cover all edge cases of concurrent shutdown and metric recording.

Testing Notes

  • Added unit tests to verify immediate shutdown and correct metric flushing.
  • Validated test runtime improvements for DynamicResources plugin.
  • Manual and automated test runs to ensure no regression in metric recording functionality.
